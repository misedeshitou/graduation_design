\section{仿真环境介绍}
本毕业设计的路径规划训练与验证采用轻量级开源强化学习环境 Sparrow 构建（开源项目原址：\url{https://github.com/XinJingHao/Sparrow-V2}）。该环境的设计理念为“麻雀虽小，五脏俱全”，不仅底层架构轻量高效，同时涵盖了丰富的物理和感知模拟。

Sparrow 基于 Pygame 框架构建，主要模拟了一个配备二维激光雷达（2D Lidar）的差速双轮/阿克曼转向式机器人，在富含动态移动与静态随机障碍物的二维平面中执行点到点（Point-to-Point）的无碰撞导航任务。机器人的主要交互框架定义如下：
\begin{itemize}
    \item \textbf{状态观测空间（Observation）}：系统输入包含传感器返回的几十条雷达离散测距点数据、机器人自身当前的线速度与角速度以及目标点相对于车体坐标系的距离和航向偏移角。
    \item \textbf{动作输出空间（Action）}：由智能体输出连续或离散的控制指令，直接映射为机器人的目标线速度和转向角速度。
    \item \textbf{任务目标（Objective）}：训练一个具备强泛化能力的智能体（Agent），在高度不确定的复杂动态随机环境中成功规避移动障碍物，以优化的路径和最短的时间安全抵达终点。
\end{itemize}

选择 Sparrow 作为本课题核心仿真平台的考量主要集中在仿真效率与高度还原的物理逻辑两个方面：

\subsection{极具优势的仿真效率与泛化能力}
\begin{enumerate}
    \item \textbf{原生支持张量向量化（Vectorization）}：有别于传统采用循环单步运行的环境，Sparrow 完全基于 PyTorch 构建了底层逻辑。借助可自定义的超参数 $N$，环境允许同时拉起成百上千个平行的独立副本地图。所有的运动学积分、雷达模拟和碰撞检测都以矩阵运算的形式在 GPU 上并行处理，这使得强化学习算法的数据采样效率呈指数级跃升。
    \item \textbf{端到端的高效数据流通（Data Flow）}：通常的物理引擎（如 Gazebo、Webots）需要在 CPU 上完成运算，再将观测结果拷贝传输至 GPU 供神经网络进行前向推理，这带来了巨大的通信开销。而 Sparrow 直接在显存内部生成深度网络所需的 Tensor 观测张量，实现了计算节点的无缝衔接。
    \item \textbf{高度的域随机化机制（Domain Randomization）}：为了跨越从仿真应用到现实物理控制的差距（Sim2Real），环境源码被注入了极强的随机化逻辑。每轮任务重置时，环境会动态且随机地初始化障碍物的数量、形状、散布及其不规则移动速度；同时还支持在雷达扫描和底盘执行中施加高斯白噪声与控制迟滞。这种复杂性赋予了所训练出的大模型出色的抗扰动泛化能力。
\end{enumerate}

\subsection{贴近底层的物理仿真逻辑}
\begin{enumerate}
    \item \textbf{惯性运动学模型模拟（Kinematics）}：该环境摒弃了指令即时响应的理想化设定，内置了一阶低通滤波器来模拟现实世界电机和轮式的物理迟滞与惯性特性。每一次状态更新的实际输出速度 $V_{\text{now}}$ 被描述为：
    \begin{equation}
        V_{\text{now}} = K \cdot V_{\text{previous}} + (1-K) \cdot V_{\text{target}}
    \end{equation}
    这说明即便输出了较大的刹车/加速控制动作 $V_{\text{target}}$，车体速度也不会发生瞬间跃变，有效倒逼算法需提前规划预留安全距离，高度吻合真实电缆机器人的电驱响应特征。
    \item \textbf{像素级 GPU 并行雷达扫描（Lidar Scan）}：传统的二维避障常常调用耗时的计算几何库来求交点。Sparrow 巧妙地采用栅格/像素级编码来映射障碍物并模拟射线。环境会将雷达中心作为原点，利用高度并行的“烟花式”探照算法，通过张量加速同时向四周扩散计算数百条射线的首个触阻距离。这极大程度上平替并复刻了真实的激光雷达扫描应用情景。
\end{enumerate}
为了极致还原真实环境中的控制延迟与电机响应物理特性，本毕业设计在实际训练中，将上述运动学滤波器系数特别设定为 $K=0.6$。这意味着当前的理想动作指令起初只有一部分会立即生效，剩余的速度增量需要逐渐克服旋转惯性，从而倒逼智能体自主学习平滑控制与提前预判。

\subsection{动作空间与执行逻辑}
针对不同的算法框架，该仿真环境支持智能体以离散（Discrete）或连续（Continuous）两种形式进行动作输出以实现路径规划。

\subsubsection{离散动作空间选择}
在离散控制模式下，动作空间被映射为 8 个预设的控制指令。这一映射表对应了机器人在不同线速度与角速度下的运动基元（见表 \ref{tab:discrete_action}）。这种设定适合如 DQN 等无法直接输出连续动作的传统强化学习算法。

\subsubsection{连续动作空间选择}
当采用能够处理连续动作空间的先进算法（如 TransSAC、SAC 等）时，智能体策略网络（Actor）将输出一个二维动作向量 $[a_1, a_2]$。通常该向量的值域被限制在 $[-1, 1]$ 之间。环境会在底层通过预设的最大速度比例尺（Continuous Scale），将该网络输出连续投影至实际底盘物理量：
\begin{itemize}
    \item \textbf{实际线速度}：$V_l = a_1 \times 50 \text{ cm/s}$
    \item \textbf{实际角速度}：$\omega = a_2 \times 2 \text{ rad/s}$
\end{itemize}

\subsection{分布式训练架构设计--Actor-Learner-Sharer模型}
\label{sec:shared_data_module}

\subsubsection{模块设计背景与目标}

在传统的单进程强化学习算法中，环境交互（数据采样）与神经网络更新（梯度下降）通常是串行执行的。这种同步阻塞的模式导致底层硬件算力无法得到充分利用，极大限制了算法的采样效率与收敛速度。因此需要分布式训练架构来解耦数据采集与模型训练的物理进程，从而实现真正意义上的并行加速。

为了解决这一问题，本课题引入了 Actor-Learner 分布式架构。在该架构下，Actor 进程与 Learner 进程在物理空间上完全解耦。为了实现跨进程、跨异构硬件的高效协作，本节设计并实现了一个高性能的共享数据中枢模块（\texttt{Shared Data Module}）。该模块主要承担两大核心职能：
\begin{enumerate}
    \item \textbf{构建高速数据管道}：接收 Actor 并发采集的交互数据，并为 Learner 提供高吞吐量的随机批次采样（Batch Sampling）。
    \item \textbf{实现异步状态同步}：作为策略网络（Policy Network）参数的寄存器，协调 Learner 的参数下发与 Actor 的参数拉取。
\end{enumerate}

\subsubsection{整体架构设计}
共享数据模块扮演了连接数据采集端与模型训练端的“中央物流中枢”角色。为了消除 I/O 瓶颈，本系统将该模块部署于主内存（RAM）中，利用共享内存机制实现进程间的极速通信。

\subsubsection{核心数据结构设计}
为了适配多进程高并发与海量深度学习张量（Tensor）的处理需求，本模块在底层数据结构上进行了如下针对性设计：

\paragraph{1. 向量化环形队列 (Vectorized Ring Buffer)} 
考虑到 Actor 采用 $N$ 个并行环境（Vectorized Environments）同步采集数据，传统的单一维度的经验池已无法满足需求。因此，本模块将经验回放池（Replay Buffer）的状态张量扩展为三维结构 $(Max\_Size, N, State\_Dim)$。

同时，该经验池采用\textbf{环形覆盖机制}（Ring Buffer）。系统通过维护一个内部指针变量（\texttt{ptr}），当经验池数据量达到最大容量约束界限时，新写入的样本将通过取模运算自动覆盖最旧的时间步数据。这在保证数据鲜度（Data Freshness）的同时，严格控制了内存的峰值占用。

\paragraph{2. 三端异构硬件自动映射 (Tri-Device Memory Mapping)} 
深度强化学习系统的算力分布往往高度异构化。本模块在初始化阶段静态定义了三个独立的计算设备标识：采集端推理设备（\texttt{A\_dvc}）、经验池存储设备（\texttt{B\_dvc}）以及训练端计算设备（\texttt{L\_dvc}）。

在执行数据入库（\texttt{add}）与数据采样（\texttt{sample}）操作时，模块底层会自动调用 PyTorch 的张量转移接口，将 CPU 演算出的环境转移矩阵无缝搬迁至 GPU 显存中，对外屏蔽了底层物理设备的寻址复杂性。

\subsubsection{并发控制与读写分离机制}
在多进程共享内存架构中，核心技术难点在于\textbf{防范读写冲突（Race Condition）}。例如，若 Learner 在 Actor 拉取模型参数的瞬间执行覆盖写入，将导致张量数据损坏，进而引发程序崩溃。

传统的系统级互斥锁（Mutex）会导致线程进入深度的阻塞态与上下文切换，造成严重的性能损耗。为此，本系统在共享数据层实现了一种轻量级的\textbf{读写分离自旋锁（Spin-Lock）}机制，其核心控制逻辑如下：

\begin{itemize}
    \item \textbf{细粒度物理隔离}：系统舍弃了全局锁，转而维护一个状态标识数组 \texttt{busy = [False, False]}。其中，索引 $0$ 代表“模型参数锁”，索引 $1$ 代表“经验池锁”。这种细粒度锁机制将经验数据的存取与模型权重的更新在物理地址层面上进行了彻底隔离。
    \item \textbf{高吞吐无冲突并行}：得益于锁的隔离机制，当 Actor 获取“经验池锁”进行数据入库操作时，Learner 可以同时获取“模型参数锁”完成策略网络的权重上载。这两个极为高频的操作能够 $100\%$ 并行发生，互不阻塞，极大提升了系统的整体吞吐量。
    \item \textbf{轮询退避防踩踏}：当双端进程竞争同一把锁（例如 Actor 和 Learner 同时尝试访问经验池）时，自旋锁机制使未获取锁的进程进入 \texttt{while True} 的高频轮询状态（休眠 $10^{-4}$ 秒级别），而非挂起等待。由于张量转移操作耗时极短，这种短效轮询机制在保证绝对数据安全的同时，将等待开销降到了最低。
\end{itemize}

\subsubsection{本节小结}
本节详细阐述了 Actor-Learner 架构中共享数据模块的设计原理与实现细节。通过构建向量化环形队列与三端硬件映射机制，保障了海量强化学习数据的高效流转；通过引入读写分离的细粒度自旋锁，在解决并发读写冲突的前提下，压榨出了异构硬件的极限吞吐能力。该模块为构建大规模、高稳定的分布式强化学习训练框架奠定了坚实的工程基础。
